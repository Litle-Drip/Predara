1. Extend settlement-review to Kalshi & Polymarket (highest leverage, you have the pieces)
Right now settlement-review.js only hits Gemini's API. You already have working adapters for Kalshi and Polymarket in adapters.js. Refactor the ticker-extraction + platform-detection logic that already exists in app.js (it detects platform from URL for the main analyzer) and reuse it here, so one endpoint routes to the right adapter, trims the resolved contract data the same way, and sends it to Claude with a platform-aware system prompt. This alone triples your addressable market for the feature.
2. Build cross-platform settlement comparison (your actual differentiator)
compare.js already extracts live outcomes/prices across Kalshi, Polymarket, Coinbase for the same event while trading is open. Extend that same matching logic to resolved markets: pull the same event's final resolution from 2+ platforms and flag disagreement. This is the one thing I called out earlier as having no real competitor — worth building even before more polish elsewhere, since it's the highest-signal feature for a serious trader/arb audience.
3. Fix the grounding problem on the AI verdict
Currently the Haiku call judges correctness purely from its own training knowledge of the real-world event — no live news lookup. For anything resolved recently or in a low-profile market, that's a real accuracy risk, and accuracy is the entire value prop of an audit tool. Add the web_search tool to that Anthropic API call (you already have the pattern for this) so the verdict is grounded in a live source, not model memory. This is probably more important than #1 or #2 if you want anyone to trust the "discrepancy" verdicts.
4. Add an evidence/audit-trail export
No submission or evidence-package workflow exists yet — it's read-only analysis. Even a simple "export this verdict + source data + timestamp as a structured record" (JSON or PDF) turns it from an interesting analysis into something someone could actually attach to a real dispute filing. Doesn't need to be fancy — this is what makes it a "tool" instead of a "toy."
